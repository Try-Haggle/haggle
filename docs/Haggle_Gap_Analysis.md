# Haggle Engine — Gap 분석 요약

**목적:** 현재 엔진과 논문 사이의 차이점 공유 및 방향 확인
**참조:** Faratin et al. (1998), Jonker et al. (2012), Jennings et al.

---

## Gap 1 — 양보를 '가격'으로 계산하냐 vs '만족도'로 계산하냐

**현재:** 가격을 시간에 따라 직접 낮춥니다.
```
1라운드: 100만원 → 2라운드: 95만원 → 3라운드: 90만원
```

**문제:** 가격 하나만 조절합니다. 상대가 배송을 더 중요하게 여겨도 엔진이 반영하지 못합니다. Haggle은 가격·배송·품질 4개 차원을 지원한다고 설계되어 있는데, 실제 양보는 가격 하나에서만 일어나고 있습니다.

**논문 방식:** 만족도 목표를 낮추고, 거기서 가격·배송·품질을 역산합니다.
```
만족도 목표: 0.90 → 0.80 → 0.70

0.80을 달성하는 조합:
  옵션 A: 가격 95만원 + 배송 5일
  옵션 B: 가격 97만원 + 배송 2일  ← 상대가 배송을 중시하면 이걸 선택
```

> 비행기 값만 깎는 게 아니라, 총 여행 만족도를 기준으로 비행기·호텔을 조율하는 것과 같습니다.

## Gap 2 — 상대방 패턴을 관찰하느냐 안 하느냐

**현재:** 상대 제안이 오면 효용을 계산하고 수락/거절/역제안만 결정합니다. 상대가 어떤 협상 스타일인지 추적하지 않습니다.

**문제:** 매 라운드를 독립적으로만 판단하므로, 상대가 보내는 신호를 읽지 못합니다.

**논문 방식(Jonker et al.):** 상대의 제안 이력을 보고 타입을 추정합니다.

| 이동 타입 | 의미 | 권장 응답 |
|-----------|------|-----------|
| CONCESSION — 내 효용↓ 상대↑ | 진심으로 양보 중 | 나도 양보 |
| SELFISH — 내 효용↑ 상대↓ | 착취 시도 | 강하게 버팀 |
| FORTUNATE — 둘 다↑ | 서로에게 좋음 | 즉시 수락 고려 |
| SILENT — 변화 없음 | 시간 끌기 | 압박 가하기 |

```
구매자 제안: 50만원 → 52만원 → 54만원

현재: 54만원이 기준에 맞냐만 판단
논문: 매 라운드 +2만원 패턴 감지 → "계속 올려줄 의향 있음" → 급하게 양보 안 해도 됨
```

> 포커에서 상대 패턴을 전혀 안 보고 내 패만 보고 베팅하는 것과, 상대 스타일을 관찰하면서 베팅하는 것의 차이입니다.

## Gap 3 — 경쟁자 수가 내 전략에 영향을 주느냐 안 주느냐

**현재:** `CompetitionContext` 필드가 있지만, 이 정보가 실제 협상 전략을 바꾸지 않습니다. 구매자가 3명이든 0명이든 판매자 AI가 동일하게 행동합니다.

**논문 방식(Faratin §4.2.1):** 경쟁자 수에 따라 양보 속도를 자동으로 조정합니다.

```
T_dynamic = μ × |경쟁자 수|² / 평균 협상 라운드 수

경쟁자 많음 → 덜 양보 (여유 있음)
경쟁자 없음 → 더 빨리 양보 (압박)
```

| 시나리오 | 판매자 행동 |
|----------|-------------|
| 구매자 5명 동시 협상 | 강하게 버팀 |
| 구매자 1명만 남음 | 빠르게 양보 |
| 구매자 전원 이탈 | 수락 임계값 낮춤 |

> 당근마켓에서 관심자가 5명이면 "좀 더 기다려볼게요", 1명만 남으면 "그냥 이 가격에 드릴게요" 하는 것과 같습니다.

## 구현 계획 (초안)

- **Gap 3:** CompetitionContext의 경쟁자 수를 β에 연결하는 수식 추가
- **Gap 1:** 양보 곡선을 가격 공간에서 효용 공간으로 이전, 이슈 역산 로직 추가
- **Gap 2:** Opponent Model 구조를 먼저 구현하고, 초기엔 보수적 추정값으로 시작 — 거래 이력이 쌓일수록 추정 정확도가 자동으로 향상됨
