## 2. Platform Data

Haggle 플랫폼이 스킬 개발자에게 제공하는 데이터 자산.

| 데이터 | 설명 | 접근 레벨 |
|--------|------|-----------|
| Market Price API | 카테고리별 평균가, 최저가, 추세 | Public |
| Transaction Patterns | 카테고리별 평균 라운드 수, 성공률, 양보 패턴 | Aggregated (익명) |
| Category Taxonomy | 표준화된 제품 분류 체계 | Public |
| Reputation Scores | 사용자 평판 점수 (본인 동의 시) | Authenticated |
| Listing Metadata | 리스팅 상태, 정보 완성도 | Authenticated |
| Negotiation Events | 실시간 이벤트 스트림 | Webhook |
| Engine Core API | `compute_utility`, `batch_evaluate` 직접 호출 | Authenticated |

## 3. 스킬 유형

### 3.1 Strategy Skill (전략 스킬)

협상 전략을 직접 생성하는 스킬. Haggle 플랫폼 데이터와 외부 데이터 소스를 결합하여 카테고리 특화 전략을 만든다.

```python
class SneakerStrategySkill(StrategySkill):
    """스니커즈 전문 협상 전략 스킬.

    Haggle Market Data + StockX + GOAT 데이터를 결합하여
    스니커즈 리셀 시장에 최적화된 MasterStrategy를 생성한다.
    """

    SKILL_ID = "sneaker-strategy-v1"
    SUPPORTED_CATEGORIES = ["sneakers", "shoes.limited_edition"]

    async def generate_strategy(
        self, listing: ListingContext, user_prefs: UserPreferences
    ) -> MasterStrategy:
        # 1) Haggle 플랫폼 데이터: 카테고리 평균가 + 거래 패턴
        market = await self.haggle_data.get_market_price(
            category="sneakers",
            item_ref=listing.title,
        )
        patterns = await self.haggle_data.get_transaction_patterns(
            category="sneakers",
        )

        # 2) 외부 데이터: StockX + GOAT 실시간 시세
        stockx_price = await self.stockx_api.get_lowest_ask(listing.sku)
        goat_price = await self.goat_api.get_lowest_price(listing.sku)
        external_avg = (stockx_price + goat_price) / 2

        # 3) 가격 범위 결정 — 플랫폼 데이터 + 외부 시세 가중 평균
        fair_value = market.avg_price * 0.4 + external_avg * 0.6
        p_target = fair_value * (1 - user_prefs.aggression * 0.15)
        p_limit = fair_value * 1.05  # 시세 대비 5%까지 허용

        # 4) 양보 곡선 파라미터 — 거래 패턴 기반
        beta = self._compute_beta(patterns.avg_rounds, patterns.concession_rate)

        return MasterStrategy(
            p_target=p_target,
            p_limit=p_limit,
            beta=beta,
            t_max=min(patterns.avg_rounds + 2, 10),
            weights=UtilityWeights(w_p=0.50, w_t=0.20, w_f=0.15, w_r=0.15),
            metadata={
                "skill": self.SKILL_ID,
                "stockx_ref": stockx_price,
                "goat_ref": goat_price,
                "haggle_market_avg": market.avg_price,
            },
        )
```

### 3.2 Data Skill (데이터 스킬)

외부 데이터를 표준 형식으로 변환하여 다른 스킬이나 Engine Core에 제공하는 스킬.

```python
class KBBAutoValuationSkill(DataSkill):
    """KBB(Kelley Blue Book) 기반 중고차 가치 평가 스킬.

    VIN 또는 차량 정보로 KBB 시세를 조회하고,
    Haggle 표준 PriceRecommendation 형식으로 반환한다.
    """

    SKILL_ID = "kbb-auto-valuation-v1"
    SUPPORTED_CATEGORIES = ["vehicles.cars", "vehicles.trucks", "vehicles.suv"]

    async def evaluate(
        self, listing: ListingContext
    ) -> PriceRecommendation:
        # 1) 차량 정보 추출
        vehicle_info = self._parse_vehicle_info(listing)

        # 2) KBB API 조회
        kbb_data = await self.kbb_api.get_valuation(
            year=vehicle_info.year,
            make=vehicle_info.make,
            model=vehicle_info.model,
            mileage=vehicle_info.mileage,
            condition=vehicle_info.condition,
            zip_code=listing.seller_location,
        )

        # 3) Haggle 플랫폼 데이터와 교차 검증
        haggle_market = await self.haggle_data.get_market_price(
            category=f"vehicles.{vehicle_info.body_type}",
            filters={"make": vehicle_info.make, "model": vehicle_info.model},
        )

        # 4) 표준 형식 반환
        return PriceRecommendation(
            fair_value=kbb_data.fair_purchase_price,
            range_low=kbb_data.fair_purchase_price * 0.92,
            range_high=kbb_data.fair_purchase_price * 1.08,
            confidence=self._calc_confidence(kbb_data, haggle_market),
            source="kbb",
            metadata={
                "kbb_condition": kbb_data.condition_grade,
                "kbb_trade_in": kbb_data.trade_in_value,
                "kbb_private_party": kbb_data.private_party_value,
                "haggle_avg": haggle_market.avg_price if haggle_market else None,
            },
        )
```

### 3.3 Interpretation Skill (해석 스킬)

규칙 기반으로 상대방 제안을 해석하는 스킬. **Interpretation Skill은 LLM 호출을 줄인다.**

```python
class AutoTradeInInterpretationSkill(InterpretationSkill):
    """중고차 트레이드인 제안 해석 스킬.

    "내 차 트레이드인하면 얼마 깎아줘?" 같은 제안을
    규칙 기반으로 해석하여 LLM 호출 없이 처리한다.

    핵심: Interpretation Skill은 LLM 호출을 줄인다.
    """

    SKILL_ID = "auto-trade-in-interpretation-v1"
    SUPPORTED_CATEGORIES = ["vehicles.*"]

    # 규칙 기반 패턴 매칭 — LLM 불필요
    TRADE_IN_PATTERNS = [
        r"트레이드[\s-]?인",
        r"trade[\s-]?in",
        r"내\s*차.*바꾸",
        r"보상[\s-]?판매",
    ]

    CONDITION_KEYWORDS = {
        "excellent": ["완벽", "무사고", "풀옵션", "excellent", "mint"],
        "good": ["양호", "깨끗", "good", "clean"],
        "fair": ["보통", "fair", "average"],
        "poor": ["수리", "사고", "poor", "damaged"],
    }

    async def interpret(
        self, proposal: RawProposal, context: NegotiationContext
    ) -> InterpretationResult:
        text = proposal.message.lower()

        # 1) 트레이드인 제안 감지 (규칙 기반)
        is_trade_in = any(
            re.search(p, text) for p in self.TRADE_IN_PATTERNS
        )
        if not is_trade_in:
            return InterpretationResult(handled=False)  # 다음 스킬로 패스

        # 2) 차량 상태 추정 (키워드 기반)
        condition = "fair"  # 기본값
        for grade, keywords in self.CONDITION_KEYWORDS.items():
            if any(kw in text for kw in keywords):
                condition = grade
                break

        # 3) KBB 트레이드인 가치 조회
        trade_in_value = await self.kbb_api.get_trade_in_value(
            description=proposal.message,
            condition=condition,
        )

        # 4) 순가격 재계산 — LLM 호출 없이 완료
        adjusted_price = context.current_offer - trade_in_value

        return InterpretationResult(
            handled=True,
            proposal_type="TRADE_IN",
            adjusted_price=adjusted_price,
            metadata={
                "trade_in_value": trade_in_value,
                "estimated_condition": condition,
                "original_offer": context.current_offer,
            },
            # Engine Core가 adjusted_price로 재계산
            recalculate_utility=True,
        )
```

## 4. Skill Interface 표준

모든 스킬은 다음 `SkillManifest`를 구현해야 한다. Protobuf 형식으로 정의.

```protobuf
message SkillManifest {
  string skill_id = 1;
  string name = 2;
  string description = 3;
  string version = 4;
  string author = 5;

  enum SkillType {
    STRATEGY = 0;
    DATA = 1;
    INTERPRETATION = 2;
    REPUTATION = 3;
    SHIPPING = 4;
  }
  SkillType type = 6;

  repeated string supported_categories = 7;
  repeated string required_permissions = 8;
  int32 max_latency_ms = 9;
  float reliability_sla = 10;
  PricingModel pricing = 11;
}

message PricingModel {
  enum Type {
    FREE = 0;
    PER_CALL = 1;
    MONTHLY = 2;
    REVENUE_SHARE = 3;
  }
  Type type = 1;
  float price = 2;
  float revenue_share_pct = 3;
}
```

## 6. 수익 모델

### 수익 분배 (확정)

| 구분 | 비율 | 설명 |
|------|------|------|
| **기본** | 거래 수수료(1.5%)의 10% | 개발자에게 0.15%p |
| **Top 3 보너스** | 카테고리별 상위 3개 스킬에 +5%p | 최대 0.225%p |

### 거래별 수익 예시 ($68 거래)

| 구분 | 계산 | 개발자 수익 |
|------|------|------------|
| 기본 | $68 x 1.5% x 10% | $0.102 |
| Top 3 | $68 x 1.5% x 15% | $0.153 |

### 월간 수익 예시 (10,000건 사용)

| 구분 | 계산 | 월 수익 |
|------|------|---------|
| 기본 | $0.102 x 10,000 | $1,020 |
| Top 3 | $0.153 x 10,000 | $1,530 |

### 랭킹 기준

| 기준 | 가중치 |
|------|--------|
| 목표가 달성률 | 30% |
| 협상 성공률 | 25% |
| 평균 라운드 수 | 15% |
| LLM 절감률 | 15% |
| 사용자 평점 | 15% |

> 카테고리별 독립 랭킹. Haggle 1st-Party 스킬은 랭킹에서 제외.

## 7. 스킬 품질 보장

### 심사 프로세스

```
자동 검증 → 샌드박스 테스트 1,000건 → 수동 리뷰 → 베타 출시 → 정식 출시
```

1. **자동 검증** — manifest 형식, 인터페이스 준수, 보안 스캔
2. **샌드박스 테스트 1,000건** — 모의 협상 시나리오로 성능/안정성 검증
3. **수동 리뷰** — Haggle 팀이 코드 품질, 비즈니스 로직 검토
4. **베타 출시** — 제한된 사용자에게 배포, 실제 환경 모니터링
5. **정식 출시** — 마켓플레이스에 공개 등록

### 모니터링 기준

| 지표 | 기준 |
|------|------|
| 응답 시간 P95 | < `max_latency_ms` (manifest에 선언한 값) |
| 에러율 | < 1% |
| 사용자 평점 | 3.5+ 유지 |

**위반 시:** 자동 비활성화 + 개발자 통지.

## 9. 구독 연동

### 구독 티어별 스킬 기능

| 기능 | Free | Premium ($9.99) | Business ($29.99) |
|------|------|-----------------|-------------------|
| 마켓플레이스 사용 | 무료 스킬만 | 전체 | 전체 |
| 스킬 만들기 | - | 노코드만 | 노코드 + 커스텀 코드 |
| 노코드 빌더 | - | 기본 템플릿 3개 | 전체 템플릿 |
| 커스텀 코드 | - | - | Python/TS |
| 외부 API 연동 | - | - | 무제한 |
| 스킬 배포/판매 | - | - | 마켓플레이스 등록 |
| 수익 분배 | - | - | 거래 수수료 10-15% |
| 샌드박스 | - | 100건/월 | 무제한 |
| 분석 대시보드 | - | 기본 | 고급 (A/B 테스트 포함) |

- **Free → Premium:** 노코드로 특화 전략을 만들면 경쟁 우위 → 업그레이드 동기
- **Premium → Business:** 스킬을 만들어 판매하면 수익 창출 가능 → 비즈니스 전환 동기
