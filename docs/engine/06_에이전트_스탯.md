# 06. 에이전트 능력치 시스템

**문서:** Haggle Engine Architecture v1.0.2 — 에이전트 능력치 시스템
**범위:** 8개 스탯 전체 사양, 파라미터 변환 공식, 가중치 자동 도출, 6개 프리셋, 예산 제약
**관련 문서:** [00_INDEX.md](./00_INDEX.md) | [02_효용_함수.md](./02_효용_함수.md) | [05_의사결정_전술.md](./05_의사결정_전술.md)

## 1. 개요

사용자가 이해하기 쉬운 8개의 **능력치(Stats)**를 통해 협상 에이전트의 성격을 정의한다. 각 스탯은 Engine Core의 수학 파라미터로 자동 변환된다.

**설계 철학:**

- **사용자 친화**: "앵커링 85" ← 직관적. "β=2.75, anchor_ratio=0.66" ← 비직관적.
- **엔진 분리**: AgentStats → 파라미터 변환은 Strategy Skill에서 수행. Engine Core는 스탯의 존재를 모른다.
- **결정론적 변환**: 동일 스탯 → 동일 파라미터. 예외 없음.
- **균형 제약**: 총합 400포인트로 만능 에이전트 방지. 강점이 있으면 약점도 있다.

```
사용자: "Tiger 프리셋으로 협상해줘"
    │
    ▼
Strategy Skill:
  AgentStats { anchoring:85, tenacity:80, resolve:60,
               marketSense:40, riskRadar:20, scrutiny:15,
               patience:55, rapport:45 }
    │
    ▼
Stats → Parameter Converter (결정론적):
  weights: {w_p:0.358, w_t:0.221, w_r:0.169, w_s:0.252}
  β: 2.62, α: 1.785
  anchor_ratio: 0.66, U_threshold: 0.58, ...
    │
    ▼
Engine Core: 일반적인 compute_utility 수행
  (Engine Core는 "Tiger"가 뭔지 모름. 파라미터만 받으면 됨.)
```

## 3. 그룹 1: 전투 스타일 (Battle Style)

### 3.1 Anchoring (앵커링) — 첫 제안 공격성

**"시장가 대비 얼마나 공격적으로 첫 제안을 하는가"**

```
엔진 파라미터 변환:
  anchor_ratio = 1.0 - (stat / 100) × 0.4
  p_target = market_average × anchor_ratio
```

| 스탯 | anchor_ratio | 시장가 $800 기준 p_target | 해석 |
|------|-------------|--------------------------|------|
| 10 (최저) | 0.96 | $768 | 시장가의 96%에서 시작. 매우 보수적. |
| 30 | 0.88 | $704 | 시장가의 88%에서 시작. |
| 50 (중립) | 0.80 | $640 | 시장가의 80%에서 시작. |
| 70 | 0.72 | $576 | 시장가의 72%에서 시작. 공격적. |
| 90 (최고) | 0.64 | $512 | 시장가의 64%에서 시작. 매우 공격적. |

**스탯 Low(10):** 시장가의 ~96%에서 시작. 상대를 자극하지 않지만, 할인 여지가 작다.
**스탯 High(90):** 시장가의 ~64%에서 시작. 높은 할인 기대. 상대가 무시할 수도 있다.

### 3.2 Tenacity (끈기) — 양보 속도

**"Faratin 곡선의 β를 결정. 양보를 얼마나 천천히 하는가"**

```
엔진 파라미터 변환:
  β = 0.3 + (stat / 100) × 2.9
```

| 스탯 | β | 곡선 형태 | 해석 |
|------|---|----------|------|
| 10 (최저) | 0.59 | 볼록 (급양보) | 초반 크게 양보. 빠른 합의 추구. |
| 30 | 1.17 | 약간 선형 | 무난한 양보 속도. |
| 50 (중립) | 1.75 | 약오목 | 양보를 조금씩 늦춤. |
| 70 | 2.33 | 오목 | 후반까지 버팀. |
| 90 (최고) | 2.91 | 강오목 (고집) | 끝까지 최소 양보. 교착 위험. |

### 3.3 Resolve (결단력) — 마지노선 엄격도

**"수락 가능 최저 효용 (U_threshold)과 목표 효용 (U_aspiration)을 결정"**

```
엔진 파라미터 변환:
  U_threshold = 0.25 + (stat / 100) × 0.55
  U_aspiration = U_threshold + 0.15
```

| 스탯 | U_threshold | U_aspiration | 해석 |
|------|-------------|-------------|------|
| 10 (최저) | 0.305 | 0.455 | 웬만하면 수락. 거래 성사율 높음. |
| 50 (중립) | 0.525 | 0.675 | 보통 기준. |
| 90 (최고) | 0.745 | 0.895 | 기준 미달 시 거래 안함. 성사율 낮음. |

## 4. 그룹 2: 분석력 (Intelligence)

### 4.1 Market Sense (시장 감각) — 시장 데이터 활용도

**"CompetitionContext를 얼마나 적극적으로 활용하는가"**

```
엔진 파라미터 변환:
  γ = (stat / 100) × 0.2
  market_utilization = stat / 100
  cross_pressure_sensitivity = stat / 100
```

| 스탯 | γ | market_utilization | 해석 |
|------|---|-------------------|------|
| 10 (최저) | 0.02 | 0.10 | 상대 가격만 봄. 경쟁 상황 무시. |
| 50 (중립) | 0.10 | 0.50 | 시장 데이터 보통 수준 활용. |
| 90 (최고) | 0.18 | 0.90 | 시장 트렌드+경쟁자+가격추이 적극 반영. |

### 4.2 Risk Radar (위험 감지) — 판매자 평판 민감도

**"V_r에서 평판(r_score)의 가중치와 최소 기준을 결정"**

```
엔진 파라미터 변환:
  w_rep = 0.3 + (stat / 100) × 0.5
  r_score_minimum = (stat / 100) × 0.6
```

| 스탯 | w_rep | r_score_minimum | 해석 |
|------|-------|-----------------|------|
| 10 (최저) | 0.35 | 0.06 | 평판 거의 무관. 누구와도 거래. |
| 50 (중립) | 0.55 | 0.30 | 평판 보통 수준 고려. |
| 90 (최고) | 0.75 | 0.54 | 고평판만 상대. 저평판 자동 필터. |

**r_score_minimum의 역할:** Batch Evaluate 시 r_score < r_score_minimum인 리스팅은 평가 전에 제외한다. Engine Core의 수학에는 영향 없음 (필터링은 Skill Layer에서 수행).

### 4.3 Scrutiny (검증력) — 리스팅 정보 요구 수준

**"V_r에서 정보 완성도(i_completeness)의 가중치와 최소 기준을 결정"**

```
엔진 파라미터 변환:
  w_info = 1 - w_rep   // Risk Radar의 w_rep에서 자동 도출
  i_completeness_minimum = (stat / 100) × 0.7
```

| 스탯 | w_info (w_rep=0.55 기준) | i_completeness_minimum | 해석 |
|------|-------------------------|------------------------|------|
| 10 (최저) | 0.45 | 0.07 | 사진 없어도 OK. 정보 부족 수용. |
| 50 (중립) | 0.45 | 0.35 | 기본 사진/설명 필요. |
| 90 (최고) | 0.45 | 0.63 | 상세 사진+스펙+설명 모두 요구. |

**주의:** w_info는 Risk Radar의 w_rep에 종속적이다 (w_info = 1 - w_rep). Scrutiny 스탯은 주로 i_completeness_minimum(사전 필터 기준)에 영향을 준다.

## 5. 그룹 3: 시간/관계 (Context)

### 5.1 Patience (인내심) — 시간 압박 저항

**"V_t의 α(시간 민감도)와 V_t_floor(시간 바닥값)를 결정"**

```
엔진 파라미터 변환:
  α = 0.3 + (stat / 100) × 2.7
  V_t_floor = (stat / 100) × 0.85
```

| 스탯 | α | V_t_floor | 해석 |
|------|---|-----------|------|
| 10 (최저) | 0.57 | 0.085 | 시간 = 강한 압박. 빨리 끝내고 싶어. |
| 50 (중립) | 1.65 | 0.425 | 적당한 여유. V_t 최소 42.5%. |
| 90 (최고) | 2.73 | 0.765 | 시간 = 거의 무의미. V_t 최소 76.5%. |

### 5.2 Rapport (관계력) — 관계 중시도

**"V_s의 V_s_base(초면 기본값)와 N_threshold(최대 신뢰 기준)를 결정"**

```
엔진 파라미터 변환:
  V_s_base = 0.3 + (stat / 100) × 0.5
  n_threshold = 15 - (stat / 100) × 10

추가 효과:
  stat > 60: late_round_aggression_modifier = 0.8
    (후반 라운드에서 양보 곡선의 공격성을 20% 완화)
```

| 스탯 | V_s_base | n_threshold | late_round_modifier | 해석 |
|------|----------|-------------|---------------------|------|
| 10 (최저) | 0.35 | 14 | 1.0 (없음) | 관계 무관. 가격만. |
| 50 (중립) | 0.55 | 10 | 1.0 (없음) | 보통 수준의 관계 중시. |
| 70 | 0.65 | 8 | 0.8 | 관계 중시. 후반 공격 완화. |
| 90 (최고) | 0.75 | 6 | 0.8 | 관계 최우선. 빠른 신뢰 구축. |

**late_round_aggression_modifier의 역할:** Faratin 곡선의 P(t) 산출 후, 최종 역제안 가격에 적용한다. modifier = 0.8이면 마지노선 방향으로의 이동폭이 20% 줄어든다 (더 관대한 역제안).

## 6. 가중치 자동 도출

스탯으로부터 UtilityWeights (w_p, w_t, w_r, w_s)를 자동 도출한다. LLM 호출 없이 결정론적으로 계산한다.

```
Step 1: 원시 가중치 계산

  raw_wp = anchoring × 0.4 + tenacity × 0.3 + resolve × 0.2 + marketSense × 0.1
  raw_wt = (100 - patience) × 0.8 + tenacity × 0.2
  raw_wr = riskRadar × 0.5 + scrutiny × 0.5
  raw_ws = rapport × 0.9 + patience × 0.1

Step 2: 정규화 (합계 = 1.0)

  total = raw_wp + raw_wt + raw_wr + raw_ws
  w_p = raw_wp / total
  w_t = raw_wt / total
  w_r = raw_wr / total
  w_s = raw_ws / total
```

**도출 논리:**

| 가중치 | 주요 기여 스탯 | 이유 |
|--------|--------------|------|
| w_p (경제) | Anchoring(40%), Tenacity(30%), Resolve(20%), MarketSense(10%) | 가격 공격성 + 양보 저항 + 기준 엄격성 = 경제적 효용 중시 |
| w_t (시간) | (100-Patience)(80%), Tenacity(20%) | 인내심 낮으면 시간 압박 크게 느낌. 끈기가 보조적으로 시간 민감도 증가 |
| w_r (리스크) | RiskRadar(50%), Scrutiny(50%) | 평판 민감 + 정보 요구 = 리스크 회피 성향 |
| w_s (관계) | Rapport(90%), Patience(10%) | 관계 중시도가 지배적. 인내심이 약간 기여 |

### v1.0.2 확장: 다중 Term 매핑

다중 Term 모드에서 AgentStats가 Term 가중치 분배에 영향:

```
Anchoring    → U_start (얼마나 공격적으로 시작하는가)
Tenacity     → beta_base (양보 형태)
Resolve      → u_threshold, u_aspiration (수락 기준)
Market Sense → 역산 전략 선택 (PROPORTIONAL vs OPPONENT_AWARE)
Risk Radar   → 리스크 관련 INFORMATIONAL Term 가중치
Scrutiny     → 품질 관련 INFORMATIONAL Term 가중치
Patience     → t_deadline 스케일링, V_t_floor
Rapport      → Opponent Model의 trust_weight (사전분포 신뢰도)
```

## 7. 예산 제약

```
총 예산: 400 포인트
스탯 수: 8
스탯당 최소: 10
스탯당 최대: 90

고정 비용: 10 × 8 = 80
자유 배분: 400 - 80 = 320

제약 조건:
  ∀ stat_i: 10 ≤ stat_i ≤ 90
  Σ stat_i = 400
```

**왜 400인가?** 평균 50 × 8 = 400. 모든 스탯을 균등하게 50으로 놓으면 정확히 400. 한 스탯을 올리면 반드시 다른 스탯이 내려가야 한다. 이것이 에이전트의 **개성**을 만든다.

**엣지 케이스:**

| 케이스 | 검증 |
|--------|------|
| 모든 스탯 = 50 | 합계 400. 유효. Fox 프리셋. |
| 한 스탯 90, 나머지 균등 | 90 + 7×(310/7) ≈ 90 + 7×44.3. 합계 400. 유효. |
| 두 스탯 90 | 180 + 6×(220/6) ≈ 180 + 6×36.7. 합계 400. 유효. |
| 합계 ≠ 400 | **오류: INVALID_STATS_BUDGET** |
| 스탯 < 10 또는 > 90 | **오류: STAT_OUT_OF_RANGE** |

## 10. 참조 구현: Stats → Engine Parameters

```python
import math

def validate_stats(stats: dict) -> dict:
    """AgentStats 유효성 검증."""
    fields = ['anchoring', 'tenacity', 'resolve', 'market_sense',
              'risk_radar', 'scrutiny', 'patience', 'rapport']

    for field in fields:
        val = stats.get(field, 0)
        if val < 10 or val > 90:
            return {'valid': False, 'error': 'STAT_OUT_OF_RANGE',
                    'detail': f'{field}={val}, must be [10, 90]'}

    total = sum(stats[f] for f in fields)
    if total != 400:
        return {'valid': False, 'error': 'INVALID_STATS_BUDGET',
                'detail': f'total={total}, must be 400'}

    return {'valid': True, 'error': None, 'total_points': total}


def stats_to_parameters(stats: dict) -> dict:
    """
    AgentStats → Engine 파라미터 결정론적 변환.
    순수 함수: 외부 I/O 없음, 동일 입력 → 동일 출력.
    """
    validation = validate_stats(stats)
    if not validation['valid']:
        raise ValueError(validation['error'] + ': ' + validation['detail'])

    a = stats['anchoring']
    te = stats['tenacity']
    re = stats['resolve']
    ms = stats['market_sense']
    rr = stats['risk_radar']
    sc = stats['scrutiny']
    pa = stats['patience']
    ra = stats['rapport']

    # --- 그룹 1: 전투 스타일 ---
    anchor_ratio = 1.0 - (a / 100) * 0.4
    beta = 0.3 + (te / 100) * 2.9
    u_threshold = 0.25 + (re / 100) * 0.55
    u_aspiration = u_threshold + 0.15

    # --- 그룹 2: 분석력 ---
    gamma = (ms / 100) * 0.2
    market_utilization = ms / 100
    cross_pressure_sensitivity = ms / 100
    w_rep = 0.3 + (rr / 100) * 0.5
    w_info = 1.0 - w_rep
    r_score_minimum = (rr / 100) * 0.6
    i_completeness_minimum = (sc / 100) * 0.7

    # --- 그룹 3: 시간/관계 ---
    alpha = 0.3 + (pa / 100) * 2.7
    v_t_floor = (pa / 100) * 0.85
    v_s_base = 0.3 + (ra / 100) * 0.5
    n_threshold = max(1, round(15 - (ra / 100) * 10))
    late_round_aggression_modifier = 0.8 if ra > 60 else 1.0

    # --- 가중치 자동 도출 ---
    raw_wp = a * 0.4 + te * 0.3 + re * 0.2 + ms * 0.1
    raw_wt = (100 - pa) * 0.8 + te * 0.2
    raw_wr = rr * 0.5 + sc * 0.5
    raw_ws = ra * 0.9 + pa * 0.1

    total = raw_wp + raw_wt + raw_wr + raw_ws
    w_p = raw_wp / total
    w_t = raw_wt / total
    w_r_weight = raw_wr / total
    w_s = raw_ws / total

    return {
        'weights': {
            'w_p': round(w_p, 4),
            'w_t': round(w_t, 4),
            'w_r': round(w_r_weight, 4),
            'w_s': round(w_s, 4),
        },
        'anchor_ratio': round(anchor_ratio, 4),
        'alpha': round(alpha, 4),
        'v_t_floor': round(v_t_floor, 4),
        'beta': round(beta, 4),
        'w_rep': round(w_rep, 4),
        'w_info': round(w_info, 4),
        'u_threshold': round(u_threshold, 4),
        'u_aspiration': round(u_aspiration, 4),
        'gamma': round(gamma, 4),
        'market_utilization': round(market_utilization, 4),
        'cross_pressure_sensitivity': round(cross_pressure_sensitivity, 4),
        'r_score_minimum': round(r_score_minimum, 4),
        'i_completeness_minimum': round(i_completeness_minimum, 4),
        'v_s_base': round(v_s_base, 4),
        'n_threshold': n_threshold,
        'late_round_aggression_modifier': late_round_aggression_modifier,
    }

```

│  AgentStats → Engine Parameters 매핑                                │
│                                                                     │
│  ┌─── 전투 스타일 ───────────────────────────────────────────────┐  │
│  │ Anchoring  ──→ anchor_ratio ──→ p_target                     │  │
│  │ Tenacity   ──→ β (양보 속도)                                 │  │
│  │ Resolve    ──→ U_threshold, U_aspiration                     │  │
│  └──────────────────────────────────────────────────────────────┘  │
│                                                                     │
│  ┌─── 분석력 ───────────────────────────────────────────────────┐  │
│  │ Market Sense ──→ γ, market_utilization, cross_pressure       │  │
│  │ Risk Radar   ──→ w_rep, r_score_minimum (사전 필터)          │  │
│  │ Scrutiny     ──→ w_info (=1-w_rep), i_completeness_minimum  │  │
│  └──────────────────────────────────────────────────────────────┘  │
│                                                                     │
│  ┌─── 시간/관계 ────────────────────────────────────────────────┐  │
