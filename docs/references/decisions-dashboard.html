▶Haggle이란?
한 줄 요약
AI가 자동으로 가격을 협상하고, 스테이블코인(USDC — 1코인 = $1인 디지털 달러)으로 즉시 결제하는 P2P 마켓플레이스 프로토콜. "The Stripe of Negotiations."

Stripe가 온라인 결제를 "카드번호 입력 → 완료"로 단순화했듯, Haggle은 중고 거래의 가격 협상을 "AI에게 맡기면 끝"으로 단순화합니다.

누구를 위한 것인가
구매자

AI가 대신 협상해서 10-20% 할인. 배송비 분담 비율도 협상 가능. 더 이상 어색한 흥정 메시지를 보낼 필요 없음.

판매자

eBay 수수료 13% 대신 Haggle 1.5%. $450 아이폰 판매 시 eBay는 ~$390 수령, Haggle은 ~$443 수령.

타겟 마켓

Apple 전자제품 리세일, US-only로 시작. 이유: 고가 + 표준화된 제품 = 협상 가치가 높고, 상태 판단이 명확함.

수익 모델

거래 수수료 1.5% + 배송 마진 (EasyPost 할인가와 사용자 청구가 차이). $450 거래 기준 건당 ~$7-8.

▶왜 크립토인가? (USDC + x402)
"왜 그냥 카드 결제 안 쓰나요?"

짧은 대답: 규제 회피 + 낮은 수수료 + 프로그래밍 가능.

1. Non-custodial = 송금업 라이선스 불필요

미국에서 다른 사람의 돈을 대신 보관하거나 전송하면 Money Transmitter License(송금업 라이선스)가 필요합니다. 50개 주 각각에서 별도 취득해야 하고, 비용만 수십만 달러입니다.

eBay 방식 = 내 돈을 eBay 은행 계좌에 맡김 (custodial). Haggle 방식 = 내 돈이 코드에 잠김, Haggle은 열쇠가 없음 (non-custodial). 마치 집 열쇠를 부동산 관리인에게 주는 것 vs 조건 충족 시 자동으로 열리는 스마트 도어록을 설치하는 것.

스마트 컨트랙트를 사용하면 Haggle이 자금에 접근할 수 없으므로 FinCEN(미국 금융범죄단속네트워크) 면제를 주장할 수 있습니다.

3. 프로그래밍 가능한 돈

스마트 컨트랙트로 "배송 완료 → 24시간 대기 → 자동 릴리즈" 같은 조건부 결제를 코드로 강제할 수 있습니다. 전통 결제에서는 이런 로직을 중앙 서버에서 관리해야 하고, 그러면 다시 custodial 문제가 생깁니다.

주요 용어 정리
USDCCircle이 발행하는 달러 연동 스테이블코인. 1 USDC = 항상 $1. 비트코인처럼 가격이 오르내리지 않음.
Base L2Coinbase가 운영하는 이더리움 레이어2 네트워크. 이더리움의 보안을 유지하면서 가스비(트랜잭션 수수료)를 ~$0.01 수준으로 낮춤.
x402Coinbase가 만든 HTTP 기반 결제 프로토콜. 웹 표준인 HTTP 402 (Payment Required) 상태 코드를 활용하여 웹 요청에 크립토 결제를 내장함.
스마트 컨트랙트블록체인 위에서 돌아가는 프로그램. 한번 배포하면 누구도 (Haggle 포함) 마음대로 바꿀 수 없고, 정해진 조건에 따라 자동으로 실행됨.

▶🚀 목표 달성 시 기업 수익 & 가치
목표: 미국 이커머스 0.1% + 중고거래 1%

미국 이커머스 전체 시장 ~$1.3조의 0.1% + 중고거래 시장 ~$2,000억의 1% = Haggle을 통해 거래되는 총 금액 $25-33억

용어 설명
GMV (Gross Merchandise Volume) = 플랫폼에서 거래된 상품의 총 금액. 예: Haggle에서 iPhone 1만 대가 각 $450에 팔리면 GMV = $4.5M. 이건 Haggle의 수입이 아니라 "얼마나 많은 거래가 일어났는가"의 지표.
ARR (Annual Recurring Revenue) = 연간 반복 수입. GMV에서 Haggle이 실제로 가져가는 돈. 수수료 1.5%면 GMV $100M → ARR $1.5M.
ARR (전체 수익) = 거래 수수료(1.5%) + 아래 부가 수익원을 모두 합한 연간 수입. 테이크레이트(실질 수수료율)가 3-5%로 올라가는 효과.
마진율 = 수입 중 실제 이익 비율. Haggle은 서버 비용 외에 원가가 거의 없으므로 96%+ (eBay, Poshmark도 비슷한 마진율).

▶확정결제 수단: x402 + USDCBase L2
설명

x402는 Coinbase가 만든 HTTP 기반 결제 프로토콜. HTTP 402 (Payment Required) 상태 코드를 활용하여 웹 요청에 결제를 내장합니다. USDC(USD Coin)는 Circle이 발행하는 달러 연동 스테이블코인 (1 USDC = $1). 가격이 변동하지 않는 "디지털 달러"입니다.

Base는 Coinbase가 운영하는 이더리움 L2(레이어2 — 이더리움 위에서 돌아가는 더 빠르고 저렴한 네트워크). 가스비(블록체인 트랜잭션 수수료)가 ~$0.01로 매우 낮아 소액 결제에 적합합니다.

왜 이걸 선택했나

기존 카드 결제(Stripe 등)는 분쟁 시 자금을 판매자로부터 강제 회수할 수 있지만(chargeback), 크립토는 non-custodial(Haggle이 자금을 직접 보관하지 않는) 구조라 스마트 컨트랙트(블록체인 위의 자동 실행 프로그램)로 보호합니다. Haggle이 직접 자금을 관리하지 않으므로 Money Transmitter(송금업) 규제를 피할 수 있습니다.

왜 $10인가

배송 트래킹 비용과 수수료의 비율 때문입니다. $10 거래에서 수수료는 $0.15 — 트래킹 비용 $0.02는 13%를 차지합니다. Apple 전자제품은 대부분 $50+ 이므로 실질적 영향은 없습니다.

$10 미만은 로컬 픽업(직거래)만 허용됩니다. 대량 판매자는 묶음 주문 총합이 $10 이상이면 허용.

▶확정컨트랙트 거버넌스: Timelock + MultisigDAO 없음
먼저: 스마트 컨트랙트가 뭔가요?

블록체인 위에서 돌아가는 프로그램입니다. 한번 배포되면 정해진 조건에 따라 자동으로 실행됩니다. 사람이 개입할 필요 없이요.

자판기를 생각하면 됩니다. 돈을 넣고 버튼을 누르면 음료가 나옵니다. 직원이 필요 없죠. 스마트 컨트랙트도 마찬가지 — "배송 완료 + 24시간 경과 → 판매자에게 돈 보내기"가 자동으로 실행됩니다.
왜 거버넌스(관리 체계)가 필요한가?

스마트 컨트랙트도 프로그램이기 때문에 때로는 업데이트가 필요합니다 — 버그 수정, 수수료 변경, 새 기능 추가 등. 문제는 "누가 이 변경을 할 수 있는가?"입니다.

한 사람이 키를 가지면위험 — 그 사람이 규칙을 마음대로 바꿀 수 있음 (최악의 경우 모든 돈을 빼갈 수 있음)
아무도 못 바꾸면위험 — 버그가 발견돼도 고칠 수 없음
여러 명이 동의해야 + 대기 시간해결책 — 한 사람의 독단 방지 + 사용자에게 확인 시간 제공
Multisig (다중 서명) 란?

Multisig = "Multi-signature" = 여러 명이 동시에 서명해야 실행되는 구조.

은행 금고를 열려면 열쇠 5개 중 3개가 필요한 것과 같습니다. 한 사람이 열쇠를 잃어버려도 금고는 안전하고, 한 사람이 악의를 가져도 나머지가 동의하지 않으면 아무것도 할 수 없습니다.

Haggle의 경우 Phase 1에서 3-of-5 구조: 5명의 서명자 중 최소 3명이 동의해야 컨트랙트 변경이 가능합니다.

Timelock (시간 잠금) 이란?

Timelock = 서명자들이 변경에 동의한 후에도 즉시 실행되지 않고 일정 시간을 기다리는 구조.

국회에서 법안이 통과돼도 "공포 후 30일"부터 시행되는 것과 비슷합니다. 이 기간 동안 시민(= Haggle 사용자)이 법안 내용을 확인하고, 마음에 안 들면 대응할 수 있습니다.

Haggle에서는 최소 48시간 대기. 이 동안 사용자는:

1. 확인무엇이 바뀌는지 블록체인에서 직접 확인 가능 (투명성)
2. 대응문제가 있으면 Settlement Contract에서 자금을 인출할 수 있음 (자기 보호)
DAO란? 그리고 왜 안 쓰나?

DAO = "Decentralized Autonomous Organization" (탈중앙 자율 조직). Multisig는 특정 5명이 결정하지만, DAO는 거버넌스 토큰을 가진 모든 사람이 투표할 수 있습니다. 주주총회와 비슷한 개념입니다.

문제: DAO에는 거버넌스 토큰이 필요합니다.

거버넌스 토큰을 만드는 것은 사실상 새로운 암호화폐를 발행하는 것입니다. 미국에서 이는 Howey Test(증권 판별 기준)에 걸릴 가능성이 높고, SEC(미국 증권거래위원회)의 규제 대상이 됩니다.

Multisig만으로도 "한 사람의 독단 방지"라는 동일한 목표를 달성할 수 있습니다. 굳이 법적 리스크를 감수할 이유가 없습니다.

▶확정Non-custodial 구조Haggle은 자금 키 미보유
Custodial vs Non-custodial이 뭔가요?

Custodial(수탁) = 회사가 당신의 돈을 대신 들고 있는 구조. 은행에 돈을 예금하거나, eBay가 구매자 대금을 보관하는 것이 여기에 해당합니다.

Non-custodial(비수탁) = 회사가 당신의 돈에 접근할 수 없는 구조. 돈은 코드(스마트 컨트랙트)에 의해 관리됩니다.

Custodial = 집 열쇠를 부동산 관리인에게 맡김. 관리인이 잘해주면 좋지만, 마음먹으면 집에 들어갈 수 있음. Non-custodial = 조건 충족 시 자동으로 열리는 스마트 도어록 설치. 아무도 열쇠를 갖고 있지 않고, 코드가 규칙을 강제함.
왜 이게 중요한가 — 규제 문제

미국에서 다른 사람의 돈을 보관하거나 전송하면 Money Transmitter License(송금업 라이선스)가 필요합니다.

Money Transmitter License의 현실:
- 50개 주 각각에서 별도 취득 필요
- 취득 비용: 수십만 달러 (주당 수천~수만 달러)
- 유지 비용: 연간 수만 달러 + 정기 감사
- 취득 기간: 6개월~2년
- 예: Coinbase는 이 라이선스를 위해 수백만 달러를 투자했음
Haggle의 Non-custodial 구조
Custodial (eBay 방식): 구매자 돈 → eBay 은행 계좌 → eBay가 판매자에게 송금 = eBay가 돈을 "들고 있음" = 규제 대상 Non-custodial (Haggle 방식): 구매자 돈 → 스마트 컨트랙트 코드에 잠김 → 조건 충족 시 자동으로 판매자에게 = Haggle은 돈에 접근 못함 = 코드가 금고
실질적 의미

Non-custodial이면 FinCEN(미국 금융범죄단속네트워크) 면제를 주장할 수 있습니다. Haggle은 자금 이동의 "규칙을 작성하는 자"이지, "돈을 만지는 자"가 아닙니다. 이 차이가 수십만 달러의 라이선스 비용과 수년의 규제 절차를 피하게 해줍니다.

▶구현완료투표 집계: Trimmed Mean결정론적 + 이상치 방어
Trimmed Mean이 뭔가요?

"극단값을 잘라낸 평균"입니다. 상위 20%와 하위 20%의 투표를 제거한 후 나머지로 평균을 계산합니다.

예시: 9명의 Reviewer가 환불 비율을 투표했습니다. 투표값: [0%, 10%, 40%, 50%, 50%, 55%, 60%, 90%, 100%] 일반 평균: 50.6% Trimmed Mean (상하위 20% 제거): 제거: 0%, 10% (하위) / 90%, 100% (상위) 남은 값: [40%, 50%, 50%, 55%, 60%] 평균: 51% → 극단적 투표(0%, 100%)가 결과에 영향을 주지 못합니다.
왜 "결정론적"(deterministic)이어야 하나?

같은 투표 데이터를 넣으면 항상 같은 결과가 나와야 합니다. 이유:

신뢰"같은 투표인데 결과가 달라졌다"면 누가 이 시스템을 믿겠습니까? 분쟁 해결 시스템에서 재현 가능성은 핵심입니다.
감사 가능누군가 결과에 이의를 제기하면, 투표 데이터로 결과를 똑같이 재현하여 검증할 수 있어야 합니다.
투명성알고리즘을 공개하면 누구나 직접 계산해서 결과를 확인할 수 있습니다. "블랙박스" 판정이 아님.

K-Means 같은 알고리즘은 랜덤 시드(초기값)에 따라 결과가 달라질 수 있어서 탈락했습니다.

왜 이상치 방어가 중요한가?

Reviewer가 악의적으로 극단값(항상 0% 또는 100%)을 투표할 수 있습니다. 일반 평균은 이런 극단값에 크게 영향받지만, Trimmed Mean은 상하위를 잘라내므로 게이밍(조작)이 어렵습니다.

▶확정투표 집계 — Trimmed Mean + K-Means fallback연속값 부분 환불
이중 모드
소규모 (<15명)5옵션 직접투표 (0/25/50/75/100%) + 가중 다수결
대규모 (≥15명)연속값 투표 (0-100%) + Trimmed Mean (상하 20% trim) + Agreement Zone
양극화 대응

Agreement Zone 합의율 < 45% → K-Means fallback (seed=hash(dispute_id), 결정론적). 중심 ±15pt 밀도 < 10%면 양극화 강제 감지.

승패 판정

결과값 >50% = 구매자 우세 (판매자 비용 부담). 부분 환불: 62% → 거래액의 62% 환불.
