# 배송_추적_모듈_설계

## 목적

이 문서는 Haggle의 배송 추적 모듈을 독립 서비스로 분리 가능한 수준으로 정의한다.
핵심 기준은 `canonical shipment state`, `shipment input SLA`, `tracking ingestion 분리`다.

## 원칙

- carrier raw status 를 그대로 domain 상태로 사용하지 않는다.
- approval 완료 이후부터 배송 입력 SLA 가 시작된다.
- tracking 수집과 사용자 표시를 분리한다.
- 배송 불이행은 trust ledger 와 dispute candidate 생성으로 이어질 수 있어야 한다.

## 선행 조건

배송 모듈은 아래 이후에 활성화된다.

- buyer approval 완료
- 필요 시 seller approval 완료
- order snapshot 고정
- shipment input due 시각 계산

## 핵심 엔티티

- `Shipment`
- `ShipmentEvent`

관련 공통 엔티티:

- `SellerApprovalPolicy`
- `SettlementApproval`
- `TrustPenaltyRecord`

## canonical 상태

- `LABEL_PENDING`
- `LABEL_CREATED`
- `IN_TRANSIT`
- `OUT_FOR_DELIVERY`
- `DELIVERED`
- `DELIVERY_EXCEPTION`
- `RETURN_IN_TRANSIT`
- `RETURNED`

## shipment input SLA

approval 완료 이후 seller 는 일정 기간 안에 배송 정보를 입력해야 한다.

예:
- `shipment_input_due_days = 2`
- `shipment_input_due_at = approved_at + 2 days`

위반 시:
- `SHIPMENT_INFO_SLA_MISSED` penalty 발행
- seller `settlement_reliability` 하락 가능
- 필요 시 dispute candidate 생성

## tracking ingestion 분리

### shipping-core
- canonical shipment status
- 상태 전이
- shipment event contract

### delivery ingest layer
- carrier webhook 수신
- raw event 정규화
- canonical event 발행

## trust 연동

배송 모듈은 seller 신뢰도에 직접 영향을 준다.

- SLA 반복 위반
- 배송 미입력
- delivery exception 반복

이 값은 결제 모듈의 settlement reliability 집계와 합쳐져
seller 의 장기 trust profile 에 반영될 수 있다.

## dispute 연동

아래 상황은 dispute 후보가 될 수 있다.

- `DELIVERY_EXCEPTION`
- `LABEL_PENDING` 장기 지속
- seller 승인 후 배송 미입력
- delivered claim 불일치

## 라벨 생성 서비스

### 개요

Haggle 은 EasyPost API 를 통해 할인된 배송 라벨을 생성한다.
판매자가 직접 우체국에서 사는 것보다 최대 60% 저렴한 상업 요율을 제공한다.

### 배송 인프라 선택: EasyPost

| 기준 | 선택 | 이유 |
|------|------|------|
| **API** | EasyPost | 100+ 캐리어 지원, TypeScript SDK, 월 고정비 없음 |
| **트래킹** | EasyPost 웹훅 | 라벨 생성 시 트래킹 무료 포함 (별도 트래킹은 $0.02/건) |
| **비용** | $0/건 (3,000건/월) → $0.08/건 (초과) | |
| **향후** | 볼륨 성장 시 USPS/UPS 직접 계약 전환 검토 | Poshmark/eBay 선례 |

### 지원 캐리어 및 접수처

| 캐리어 | 접수 가능 장소 |
|--------|-------------|
| USPS | 우체국(31,000+), Blue Box, Walgreens, Staples, 무료 집 앞 픽업 |
| UPS | UPS Store(5,000+), UPS Drop Box, CVS |
| FedEx | FedEx Office, FedEx Drop Box, Walgreens, Dollar General |

### 배송비 모델: 할인 배송 (Option C)

EasyPost 상업 요율을 기반으로 소매가보다 저렴한 배송비를 제공한다.
배송비 차액에서 소폭 마진을 확보하되, 핵심 목적은 사용자 유입이다.

```
EasyPost 상업 요율:  $4.50 (1lb USPS Ground, 예시)
소매가:              $7-8
Haggle 제시 가격:    $4.99
```

### 배송비 = 협상 대상

배송비는 고정이 아니다. 구매자와 판매자가 협상으로 분담 비율을 결정한다.

```
협상 가능 항목:
  - 상품 가격
  - 배송비 분담 비율 (0~100% 구매자/판매자)
  - 배송 방법 선택 (USPS, UPS, FedEx)

예시:
  "iPhone $430, 배송비 내가 낼게"
  "iPhone $420으로 해주면 배송비 내가 부담"
  "$450에 무료 배송"
```

### 이행 방법 (Fulfillment)

배송은 **Haggle 라벨만 허용**한다. 판매자가 자체 운송장을 입력하는 방식은 지원하지 않는다.

| 이행 방법 | 설명 |
|-----------|------|
| **Haggle 라벨** | EasyPost 통해 생성. 트래킹 무료, 무게 버퍼, APV 조정 자동 |
| **로컬 픽업** | 직거래. 배송 없음, 버퍼 없음, 최소 거래 금액 제한 없음 |

자체 배송(판매자 운송장 직접 입력)은 현재 지원하지 않는다.
이유:
- 외부 운송장 트래킹 비용 $0.02/건 발생 (Haggle 라벨은 무료)
- 무게 버퍼/APV 조정 불가
- Payment Protection 약화

> 자체 배송 허용 여부는 추후 팀 논의 안건으로 보류.

### 최소 거래 금액

| 거래 금액 | 정책 |
|-----------|------|
| $10 미만 | 로컬 픽업만 허용 |
| $10-$25 | 배송 가능, "로컬 픽업 추천" 안내 |
| $25 이상 | 일반 배송 |

Apple 전자제품은 대부분 $50+ 이므로 실질적 영향 없음.
대량 판매자는 묶음 주문 총합이 $10 이상이면 허용.

### 무게 조정 (USPS APV) 대응

USPS 는 모든 패키지를 자동 스캔하여 실제 무게와 신고 무게를 비교한다.
불일치 시 차액이 청구된다 (대부분 배송 후 10일 이내, 최대 30일).

**무게 비례 버퍼 (Conditional Hold):**

판매자가 입력한 무게의 한 단계 위 요금과의 차이를 버퍼로 Settlement Contract 에 잡는다.

```
판매자 입력: 2lb → 배송비 $7.50
한 단계 위:  3lb → 배송비 $9.00
버퍼:        $1.50

판매자 입력: 5lb → 배송비 $11.00
한 단계 위:  7lb → 배송비 $14.00
버퍼:        $3.00
```

물건이 무거울수록 버퍼도 커져서 무게 조정을 거의 항상 커버한다.

**판매자 고지 의무:**
- 라벨 생성 시 "실제 무게와 다를 경우 차액이 청구됩니다" 명시
- 무게 조정은 판매자 책임 (잘못 입력한 건 판매자 과실)

### 2단계 정산

상품 대금과 배송비 정산을 분리한다.

```
Settlement Contract 구성:
  ├── 상품 대금: $450
  ├── 배송비: $7.50 (EasyPost 비용)
  ├── 무게 버퍼: $1.50 (한 단계 위 요금 차이)
  └── Haggle 수수료: $6.75

Phase 1 — 배송 완료 + 구매자 확인 (24시간):
  구매자가 "확인 완료" 버튼 클릭 → 즉시 릴리즈
  24시간 내 미확인 → 자동 릴리즈
  24시간 내 분쟁 제기 → 릴리즈 중단, 분쟁 해결까지 보류

Phase 2 — 무게 조정 확정 (배송 완료 후 14일):
  무게 조정 없음 → 버퍼 $1.50 → 판매자 반환
  무게 조정 $1.00 → 버퍼에서 차감, 잔액 $0.50 → 판매자 반환
  무게 조정 $1.50 이상 → 버퍼 전액 사용, 초과분은 판매자 신뢰 기록에 반영
```

**정산 타임라인:**

```
Day 0:     거래 성사 → Settlement Contract (상품 + 배송비 + 버퍼)
Day 0-1:   판매자 라벨 프린트 → 패키지 접수
Day 2-5:   배송 중 (USPS Ground Advantage 기준)
Day 5:     배송 완료 (DELIVERED) → 구매자 검토 시작 (24시간)
Day 5:     구매자 "확인 완료" 클릭 → 상품 대금 즉시 릴리즈
           또는 24시간 내 분쟁 제기 → 릴리즈 중단
Day 6:     24시간 미확인 시 → 상품 대금 자동 릴리즈
Day 5-19:  무게 조정 대기 (USPS APV)
Day 19:    버퍼 정산 완료 → 잔액 판매자 반환
```

### non-custodial Settlement Contract 호환

모든 자금은 스마트 컨트랙트(Settlement Contract)에 있고 Haggle 은 키를 보유하지 않는다.

```
Settlement Contract (스마트 컨트랙트):
  ├── 상품 대금: 배송 확인 + 구매자 확인 → 판매자에게 자동 릴리즈
  ├── 배송비: EasyPost 결제 주소로 자동 전송
  ├── 버퍼: 14일 후 무게 조정 차감 → 잔액 판매자 반환
  └── emergency exit: Timelock 기간 중 사용자 직접 인출 가능
```

### EasyPost 리스크 및 대응

| 리스크 | 대응 |
|--------|------|
| 무게 조정 추가 청구 | 무게 비례 버퍼 + 판매자 고지 |
| EasyPost 서비스 장애 | MockCarrierAdapter 폴백 (판매자 직접 배송) |
| 보험 클레임 거부 사례 | MVP 에서는 EasyPost 보험 미제공, 분쟁 시스템으로 대응 |
| Trustpilot 1.7/5 평가 | 대부분 B2C 사용자 불만, API 통합에서는 안정적 |
| 볼륨 증가 시 비용 | 월 3,000건 초과 시 $0.08/건 → 볼륨 커지면 USPS 직접 계약 전환 |
