## 1. 현재 문제

6-Stage 파이프라인(Doc 26)이 매 라운드 풍부한 데이터를 생성하지만, 파이프라인 밖으로 나가는 출구가 없다.

| 데이터 | 엔진 내 존재 | 세션 종료 후 | 해자 가치 |
|--------|------------|-----------|---------|
| 최종 합의가/거절 | ✅ | ✅ DB 저장 | 낮음 (eBay도 있음) |
| 라운드별 가격 궤적 (PT) | ✅ Living Memo | ❌ 소멸 | **최고** |
| 라운드별 조건 교환 (CL) | ✅ Living Memo | ❌ 소멸 | **최고** |
| 상대 양보 패턴 (OM) | ✅ Session Memory | ❌ 소멸 | **최고** |
| 전술 사용/성공 여부 | ✅ tactic_used | ❌ 집계 없음 | **매우 높음** |
| Referee 개입 기록 | ✅ 함수 리턴값 | ❌ 로그 없음 | 높음 |
| Coach 추천가 vs 실제 차이 | ✅ 계산됨 | ❌ 기록 없음 | 높음 |
| 카테고리별 평균 할인율 | ❌ 집계 없음 | ❌ | **최고** |

**"최고"와 "매우 높음"으로 표시된 데이터가 전부 영속화되지 않고 있다.**

```
  Memo 갱신 → Phase 전이 → DB 저장 → 이벤트 발행

  Memo 갱신 → Phase 전이 → DB 저장 → 이벤트 발행

                          RoundRecord 확장 필드 저장 (매 라운드)
                                    ↓
                          세션 종료 감지 시:
                                    ↓

                                    ↓
                          NIG 집계 큐에 발행
```

## 5. 두 가지 집계 — 공개 vs 비공개

### 5.1 공개 집계 (Haggle Index — 외부 공개 가능)

카테고리 수준 통계만. 개인 식별 불가.

```
iPhone 15 Pro / Like New:
  평균 할인율:        11.3%
  평균 합의 라운드:    4.2
  평균 협상 시간:      47분
  가장 효과적인 전술:  reciprocal_concession (성사율 +18%)
  성사율 높은 시간대:  토요일 오전 10-12시
  가격대별 성사율:     $800-1000 → 42%, $1000-1200 → 31%
```

용도: 분기별 퍼블릭 리포트, Intelligence API, 언론 인용, 데이터 권위 구축.

### 5.2 비공개 집계 (대행 엔진 전용 — 절대 외부 미공개)

개인 패턴 데이터. Haggle 대행 엔진만 접근.

```
판매자 X (익명 ID):
  양보 패턴:          CONCEDER (β ≈ 0.4)
  평균 양보율:        라운드당 2.1%
  추정 마지노선:      시작가의 -15%
  조건 교환 성향:      배송비 양보에 민감
  평균 수락 라운드:    5.3
  시간대 패턴:        저녁에 더 빠르게 양보
```

용도: Coach의 L5 Signals 주입, 대행 모드 품질 향상. **이 데이터가 "Haggle 대행이 직접 하는 것보다 나은 이유"의 핵심 원천.**

## 6. 데이터 축적 타임라인

| 거래 수 | 가능해지는 것 | 데이터 자산 가치 |
|--------|-------------|---------------|
| 100건 | 기본 패턴 확인 | 없음 |
| 1,000건 | 카테고리별 초기 통계 | 낮음 |
| 10,000건 | 전술 성공률 유의미, Intelligence API 베타 | 중간 |
| 100,000건 | 합의 확률 예측 모델 학습 가능 | **자산 시작** |
| 1,000,000건 | 개인화 추천, 크로스카테고리 패턴 | **복제 불가능** |

## 7. 이게 해자인 이유

**대기업이 복제하려면:**

eBay는 최종 판매가만 안다. 과정이 없다. FB Marketplace는 메신저 대화가 비정형 텍스트라 구조화 불가. Mercari, Poshmark, OfferUp 전부 마찬가지 — 거래 결과는 있지만 "어떤 과정을 거쳐서 이 가격에 도달했는지"는 아무도 모른다.

Haggle의 6-Stage 파이프라인은 **모든 메시지가 엔진을 통과하면서 자동으로 구조화**된다. 이건 사후에 만들 수 없다. 1건부터 기록해야 100만 건째에 해자가 된다.

**경쟁자가 오늘 시작해도:** 같은 양의 데이터를 쌓으려면 같은 시간이 걸린다. 데이터는 시간을 살 수 없다.
