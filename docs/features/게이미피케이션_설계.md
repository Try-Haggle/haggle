# 게이미피케이션 설계 v3

> 거래 활동을 통해 고유한 에이전트 캐릭터를 수집하고 성장시키는 시스템.
> MVP 범위: 뽑기 + 프로필 전시 + 레벨 시스템

**구현 상태:** ❌ 미구현 (Post-MVP). 코드 0줄.
**선행 조건:** trust-core, dispute-core 완성 (✅) → DB/API Phase 3 완료 후 착수 가능

## 1. 핵심 컨셉

```
거래 완료 → 캐릭터 뽑기 (무료 보상, 1회)
캐릭터는 마스크 기반 레이어 합성으로 자동 생성
프로필에 CSS 애니메이션으로 전시
레벨과 Trust로 해금 요소 증가
```

**스택: Python (마스크 색칠) + CSS (애니메이션) + DB (저장)**
**라이브러리 0, .riv 없음, NFT 없음 (MVP)**

## 4. 배심원 별 (⭐1~5)

### 점수 산출 (7개 입력)

| 입력 | 가중치 |
|------|--------|
| Agreement Zone 진입률 | 30% |
| 결과 근접도 | 25% |
| 참여율 | 15% |
| 응답 속도 | 10% |
| 누적 참여 | 10% |
| 카테고리 다양성 | 5% |
| 고액 경험 | 5% |

### 별 매핑

| 별 | 점수 | 투표 가중치 |
|---|------|-----------|
| ⭐ | 0-30 | 0.63 |
| ⭐⭐ | 31-50 | 0.85 |
| ⭐⭐⭐ | 51-70 | 1.10 |
| ⭐⭐⭐⭐ | 71-85 | 1.45 |
| ⭐⭐⭐⭐⭐ | 86+ | 2.00 |

### 미활동 가속 감소

```
decay = 점수 × (경과일/30)² × 0.02

30일: -2%     (거의 안 느낌)
60일: -8%     (슬슬 느낌)
90일: -18%    (확 떨어짐)
120일: -32%   (2단계 강등)
180일: -72%   (거의 리셋)

1건 판정하면 타이머 리셋.
```

### 전문가 태그

```
획득: 해당 카테고리 최근 50건 중 85%+ (43/50 맞춰야)
유지: 최근 50건 중 80%+ (40/50) + 90일 이내 활동
제거: 80% 미만 OR 90일 미활동

전체 배심원 중 상위 1.4%만 전문가.
```

### 카테고리별 별 (자세히 보기)

```
전체: ⭐⭐⭐⭐ (78점)

카테고리별 (최근 50건 기준):
  Electronics  ⭐⭐⭐⭐⭐  87% (50/50건) 🏷️ 전문가
  Vehicles     ⭐⭐⭐⭐   76% (50/50건) 🏷️ 전문가
  Clothing     ⭐⭐⭐    68% (32/50건)
```

## 6. 캐릭터 등급 & 확률

### 등급 (6단계)

| 등급 | 테두리 | 특징 |
|------|--------|------|
| ⬜ Common | 흰색 | 기본 색상 |
| 🟩 Uncommon | 초록 | 약간 특별한 색 조합 |
| 🟦 Rare | 파란 | + 무늬 (줄무늬/물방울/별) |
| 🟪 Epic | 보라 | 특수 색상 (홀로그램/네온/메탈릭) |
| 🟨 Legendary | 금 | 전설 전용 디자인 (투명/황금/우주) |
| 🔴 Mythic | 홀로그램 | 종족당 1~3종, 신화적 존재 (구미호, 천룡 등) |

### 확률 — 거래액 연속 함수 + 동적 감소

```
rate(amount, total_minted) = base × ln(1 + amount/100) / sqrt(1 + total_minted/5000)
```

**거래액별 확률 (초기, 총 민팅 1,000건):**

| 등급 | base | $100 거래 | $1,000 거래 | $10,000 거래 |
|------|------|---------|-----------|------------|
| Common | 나머지 | ~75% | ~65% | ~55% |
| Uncommon | 0.15 | ~15% | ~17% | ~18% |
| Rare | 0.06 | ~7% | ~10% | ~13% |
| Epic | 0.02 | ~2.5% | ~5% | ~8% |
| Legendary | 0.003 | ~0.4% | ~2% | ~4.5% |
| Mythic | 0.0002 | ~0.03% | ~0.1% | ~0.5% |

**동적 감소로 스케일 시 자동 희소해짐. 전설 남은 개수에 따라 추가 감소.**

### 조합 수

| 등급 | 조합 수 (종족당) | × 8종족 |
|------|--------------|--------|
| Common | ~8,400만 | ~6.7억 |
| Uncommon | ~8,400만 | ~6.7억 |
| Rare | + 무늬 240종 | ~1,613억 |
| Epic | ~52만 | ~414만 |
| Legendary | 수백 | ~수천 |
| Mythic | 1~3 | **24** |

## 7. 뽑기 시스템

### 뽑기권 획득

| 트리거 | 유형 | 빈도 |
|--------|------|------|
| 거래 완료 | 일반 뽑기 1회 | 매 거래 (금액에 따라 확률 시프트) |
| 새 파라미터 조합 첫 사용 | 보너스 뽑기 1회 | 조합당 1회 (상위 등급 확률 ↑) |
| 새 카테고리 첫 거래 | 종족 확정 뽑기 | 카테고리당 1회 |
| 거래 마일스톤 | 마일스톤 뽑기 | 마일스톤마다 (에픽 확률 ↑) |
| 배심원 판정 10건 | 배심원 전용 뽑기 | 10건마다 |

### 동적 희소성

```
legendary_rate = base × ln(1 + amount/100) / sqrt(1 + total/5000) × (remaining/max)

→ 거래액 높으면 확률 ↑
→ 총 민팅 늘면 확률 ↓
→ 해당 전설 다 나오면 확률 → 0
```

```python
    masks = load_masks(species)
```

## 11. 법적 구조

- 캐릭터 뽑기: **거래 보상** (가챠 아님) → FTC 대상 아님
- 확률 공개: **필수** (한국/FTC 대비)
- MVP에서 NFT 없음: DB 기반 → 규제 대상 아님
- 추후 Soulbound: "활동 보상 기념품" → 증권 아님
- 추후 마켓: 스마트 컨트랙트 P2P → Money Transmitter 아님
- 용어: **"NFT"라 부르지 않음** → "Digital Collectibles" 또는 "에이전트 캐릭터"
